## Quick Install

```bash
# 1. Extract the bundle
unzip ai-ops-command-center-v1.0.0.zip
cd ai-ops-command-center

# 2. Make everything executable
chmod +x *.sh

# 3. Run the setup wizard
./setup-ai-apis.sh
```

That's it. The wizard will guide you through API key configuration.

## Step-by-Step Installation

### Step 1: Verify System Compatibility

```bash
# Check macOS version
sw_vers
# Expected: ProductVersion: 12.x or later

# Check bash version
bash --version | head -1
# Expected: GNU bash, version 5.x.x

# Check available disk space
df -h . | tail -1
# Recommended: 500MB+ free
```

### Step 2: Extract the Bundle

```bash
# If downloaded as ZIP
unzip ai-ops-command-center-v1.0.0.zip
cd ai-ops-command-center

# If cloned from Git
git clone https://example.com/ai-ops-command-center.git
cd ai-ops-command-center
```

### Step 3: Set Permissions

```bash
# Make all scripts executable
chmod +x *.sh

# Verify
ls -la *.sh
# All should show -rwxr-xr-x or similar
```

### Step 4: Run the Setup Wizard

```bash
./setup-ai-apis.sh
```

The wizard will:

1. **Detect existing configurations** — Checks for existing `.env` files, API keys in shell configs
2. **Guide you through each provider** — One at a time, with validation
3. **Test connectivity** — Confirms each API key works against the live endpoint
4. **Generate your `.env` file** — All keys in one place, properly formatted
5. **Run the test suite** — Validates every integration end-to-end

### Step 5: Verify Installation

```bash
# Run diagnostics
./ai-cli-diagnostics.sh

# Expected output: All APIs showing ✅ HTTP 200

# Launch the unified menu
./ai_unified_menu.sh

# Expected: Color-coded interactive menu
```

## API Key Setup

### Getting Your API Keys

#### OpenAI

1. Go to [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
2. Click **Create new secret key**
3. Copy the key (starts with `sk-`)
4. The setup wizard will validate it automatically

#### Anthropic (Claude)

1. Go to [console.anthropic.com](https://console.anthropic.com)
2. Navigate to **API Keys**
3. Create a new key (starts with `sk-ant-`)
4. Paste it when prompted by the setup wizard

#### Groq

1. Go to [console.groq.com/keys](https://console.groq.com/keys)
2. Create API key
3. Copy the key
4. The wizard will test connectivity

#### xAI / Grok

1. Go to [console.x.ai](https://console.x.ai)
2. Generate API key (starts with `xai-`)
3. The `grok_integration.sh` script will configure it

#### Google (Gemini)

1. Go to [aistudio.google.com/apikey](https://aistudio.google.com/apikey)
2. Create API key
3. Paste during setup

#### Cohere

1. Go to [dashboard.cohere.com/api-keys](https://dashboard.cohere.com/api-keys)
2. Copy your trial or production key
3. The wizard will validate it

### Manual `.env` Configuration

If you prefer to configure manually, create a `.env` file in the bundle directory:

```bash
# AI Ops Command Center — Environment Configuration
# Generated by setup-ai-apis.sh

# OpenAI
OPENAI_API_KEY=sk-your-key-here

# Anthropic
ANTHROPIC_API_KEY=sk-ant-your-key-here

# Groq
GROQ_API_KEY=gsk-your-key-here

# xAI / Grok
GROK_API_KEY=xai-your-key-here
XAI_API_KEY=xai-your-key-here

# Google / Gemini
GOOGLE_API_KEY=AIza-your-key-here

# Cohere
COHERE_API_KEY=your-key-here

# Optional: Default model preferences
DEFAULT_OPENAI_MODEL=gpt-4o
DEFAULT_ANTHROPIC_MODEL=claude-sonnet-4-20250514
DEFAULT_GROQ_MODEL=llama-3.3-70b-versatile
DEFAULT_OLLAMA_MODEL=llama3.1:8b
```

Then load it:

```bash
set -a
source .env
set +a
```

## Troubleshooting

### "Permission denied" errors

```bash
chmod +x *.sh
```

### "bash: command not found" on macOS

macOS ships with bash 3.x by default. Install bash 5+:

```bash
brew install bash
# Then use the new bash path:
/usr/local/bin/bash ./setup-ai-apis.sh
# or
/opt/homebrew/bin/bash ./setup-ai-apis.sh  # Apple Silicon
```

### API key validation fails

1. Verify the key is correct (no extra spaces or quotes)
2. Check that your account has API access enabled
3. Test connectivity manually:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" \
     -H "Authorization: Bearer $OPENAI_API_KEY" \
     https://api.openai.com/v1/models
   # Expected: 200
   ```

### Ollama not found

```bash
brew install ollama
brew services start ollama
```

### Watchdog won't start

Ensure `bc` is installed:

```bash
brew install bc
```

### MCP server connection refused

1. Verify the MCP server is running:
   ```bash
   ./mcp-process-manager.sh status
   ```
2. Check the server's port is not blocked:
   ```bash
   lsof -i :PORT_NUMBER
   ```
3. Restart the server:
   ```bash
   ./mcp-lifecycle-manager.sh restart server-name
   ```

## Post-Install Configuration

### Set Up Auto-Start Watchdog

Add to your `~/.zshrc` or `~/.bashrc`:

```bash
# Start AI watchdog on shell login
/Users/steven/scripts/bundles/ai-ops-command-center/ai-watchdog.sh run &
```

### Configure Resource Limits

Edit `ai-resource-manager.sh` to adjust limits:

```bash
MAX_MEMORY_PER_TOOL=4096    # MB — increase for heavy models
MAX_CPU_PER_TOOL=80         # Percentage
MAX_CONCURRENT_TOOLS=3      # Simultaneous tools
```

### Add to PATH (Optional)

```bash
# Add to ~/.zshrc
export PATH="/Users/steven/scripts/bundles/ai-ops-command-center:$PATH"

# Then use from anywhere:
ai-unified-menu
ai-diagnostics
```

### Environment Integration

If you use the `.env.d/` system (common in advanced setups):

```bash
# Link your .env to .env.d/
ln -s /path/to/bundle/.env ~/.env.d/ai-ops.env
```

## Uninstall

To remove AI Ops Command Center:

```bash
# Remove the bundle directory
rm -rf /path/to/ai-ops-command-center

# Remove from PATH if added
# Edit ~/.zshrc and remove the export PATH line

# Remove watchdog from auto-start
# Edit ~/.zshrc and remove the ai-watchdog.sh line

# Optionally remove API keys from .env
rm /path/to/ai-ops-command-center/.env
```
